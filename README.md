# Fullstack-приложение: Система управления товарами с RBAC

Учебный проект, выполненный в рамках практических занятий №7–12 по дисциплине «Фронтенд и бэкенд разработка». Приложение представляет собой систему с авторизацией, разграничением прав доступа и управлением каталогом товаров.

## 🚀 Технологический стек
- **Frontend**: React.js, React Router, Axios
- **Backend**: Node.js, Express, JWT, Bcrypt
- **Стилизация**: Custom CSS (Apple/Glassmorphism style)

## 📋 Выполненные задачи по модулям

### 🔐 Аутентификация и безопасность (Пр. №7–9)
- Реализована **регистрация и вход** с хешированием паролей через `bcrypt`.
- Система **двойных токенов**: 
  - `Access Token` (короткоживущий) для авторизации запросов.
  - `Refresh Token` (долгоживущий) для автоматического продления сессии.
- Защищенный маршрут `/api/auth/me` для получения данных текущего сеанса.

### 🌐 Фронтенд-интеграция (Пр. №10)
- Хранение токенов в `localStorage`.
- Настроен **Axios Interceptor**: автоматически подставляет заголовок `Authorization` и обрабатывает ошибку 401 (автопродление токена через refresh).

### 🛡️ Ролевая модель доступа (RBAC) (Пр. №11)
Реализовано три уровня доступа:
- **User**: Просмотр списка товаров и деталей.
- **Seller**: Все права пользователя + создание, редактирование и генерация тестовых товаров.
- **Admin**: Полный доступ + удаление товаров и управление списком пользователей.

## 🛠️ Основные эндпоинты API
- `POST /api/auth/register` — Регистрация
- `POST /api/auth/login` — Вход
- `GET /api/products` — Получить все товары
- `POST /api/products` — Создать товар (Seller/Admin)
- `PUT /api/products/:id` — Редактировать (Seller/Admin)
- `DELETE /api/products/:id` — Удалить (Admin)
- `GET /api/users` — Список пользователей (Admin)

## 📦 Запуск проекта
1. **Сервер**: `node app.js` (порт 8000)
2. **Клиент**: `npm start` (порт 3000)

**Тестовый аккаунт администратора:** Логин: `admin` | Пароль: `admin123`
